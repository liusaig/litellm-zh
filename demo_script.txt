# 企业级 AI 模型管理平台 - 演示脚本

> 演示时长：约 8-10 分钟
> 演示目标：展示完整的用户管理、团队管理、密钥管理、消费追踪和账单导出功能

---

## 演示脚本

【开场介绍】（约 1 分钟）

  演示者：各位好，今天我为大家演示我们企业级 AI 模型管理平台的核心功能。
  这个平台可以帮助企业统一管理多个 AI 模型的访问权限、控制预算和使用成本。

---

演示内容1：登录平台（约 1 分钟）

  演示者：首先，我来演示如何登录平台。

  操作：
  1. 打开浏览器，访问：http://localhost:4000/login
  2. 输入管理员账号和密码
  3. 点击"登录"按钮

  演示者：登录成功后，我们进入平台的主控台界面。左侧是功能导航菜单，
  可以看到我们有API密钥、用量分析、分组管理、用户管理等多个功能模块。

---

演示内容2：创建和设置分组、用户（约 2.5 分钟）

  演示者：接下来，我来创建一个新的业务分组，用于管理"研发部门"的资源使用。

  操作（创建分组）：
  1. 点击左侧菜单「分组」，跳转到：http://localhost:4000/teams
  2. 点击「创建分组」按钮
  3. 填写分组信息：
     - 分组名称：研发部门
     - 分组预算：100 美元
     - 预算周期：月
  4. 点击「确认创建」

  演示者：分组创建成功，该分组当前剩余预算为 100 美元。接下来，我们为分组创建用户。

  操作（创建用户）：
  1. 点击左侧菜单「用户」，跳转到：http://localhost:4000/users
  2. 点击「创建用户」按钮
  3. 填写用户信息：
     - 用户名：zhangsan
     - 用户角色：普通用户
     - 所属分组：研发部门
     - 个人预算：20 美元
  4. 点击「确认创建」
  5. 重复步骤创建第二个用户：lisi，个人预算设置为 30 美元

  演示者：用户创建成功，我们已经为研发部门配置了两个用户，
  每个用户都有独立的预算限制。

---

演示内容3：设置密钥，并分配给不同分组、用户使用，同时设置使用余额（约 1.5 分钟）

  演示者：现在，我为这两个用户生成 API 访问密钥，这样他们就可以调用 AI 模型接口了。

  操作：
  1. 点击左侧菜单「API密钥」，跳转到：http://localhost:4000/virtual-keys
  2. 点击「生成密钥」按钮
  3. 为用户 zhangsan 创建密钥：
     - 密钥别名：zhangsan-dev-key
     - 关联用户：zhangsan
     - 关联分组：研发部门
     - 预算限制：20 美元
     - 模型选择：全部模型
     - 有效期：永久
  4. 点击「生成」
  5. 复制生成的 API Key（例如：sk-xxxxxxxxxxxxx）
  6. 重复步骤为 lisi 生成密钥，预算设置为 30 美元

  演示者：密钥生成成功，每个用户都有独立的 API Key 和预算限制。
  分组剩余预算为 100 - 20 - 30 = 50 美元，分组内的其他用户还可以使用这笔预算。

---

演示内容4：根据密钥在客户端通过 OpenAI 标准接口成功调用 4 次不同模型的 API（约 1.5 分钟）

  演示者：现在，我来演示客户端如何通过标准 OpenAI 接口调用不同模型的 API。
  这里我会调用 4 次不同模型的接口。

  操作（打开终端或网页测试工具）：

  演示代码 1 - 调用 GPT-4o 模型：
  curl -X POST 'http://localhost:4000/chat/completions' \
    -H 'Authorization: Bearer sk-zhangsan-key' \
    -H 'Content-Type: application/json' \
    -d '{
      "model": "gpt-4o",
      "messages": [{"role": "user", "content": "你好，请用一句话介绍你自己"}]
    }'

  演示代码 2 - 调用 Claude 3.5 Sonnet 模型：
  curl -X POST 'http://localhost:4000/chat/completions' \
    -H 'Authorization: Bearer sk-zhangsan-key' \
    -H 'Content-Type: application/json' \
    -d '{
      "model": "claude-3-5-sonnet",
      "messages": [{"role": "user", "content": "什么是人工智能？"}]
    }'

  演示代码 3 - 调用 Gemini Pro 模型：
  curl -X POST 'http://localhost:4000/chat/completions' \
    -H 'Authorization: Bearer sk-zhangsan-key' \
    -H 'Content-Type: application/json' \
    -d '{
      "model": "gemini-pro",
      "messages": [{"role": "user", "content": "写一首关于春天的诗"}]
    }'

  演示代码 4 - 调用 Mistral 模型：
  curl -X POST 'http://localhost:4000/chat/completions' \
    -H 'Authorization: Bearer sk-zhangsan-key' \
    -H 'Content-Type: application/json' \
    -d '{
      "model": "mistral-large",
      "messages": [{"role": "user", "content": "1+1等于几？"}]
    }'

  演示者：大家可以看到，四次调用都成功了，返回了不同模型生成的响应。
  我们的平台完全兼容 OpenAI 的标准接口格式，用户可以无缝切换不同的模型。

---

演示内容5：展示记账界面，体现 API 调用后的余额实时变化（约 1 分钟）

  演示者：现在，让我们来看一下刚才这 4 次 API 调用的消费情况。

  操作：
  1. 点击左侧菜单「用量分析」，跳转到：http://localhost:4000/usage
  2. 选择「实时视图」或「今日视图」
  3. 展示以下数据：
     - 总消费金额（实时更新）
     - 总请求次数
     - 成功请求数 / 失败请求数
     - Token 用量统计
     - 按模型分类的消费明细

  演示者：从数据可以看到，刚才 4 次调用产生了对应的消费金额，
  这里的消费是在调用完成后实时扣除的。我们还可以点击左侧菜单「请求日志」
  http://localhost:4000/logs 查看每一次调用的详细记录。

---

演示内容6：展示按周、月导出账单功能（约 1 分钟）

  演示者：最后，我来演示如何按时间范围导出账单，方便财务部门进行费用统计。

  操作：
  1. 确保在用量分析页面：http://localhost:4000/usage
  2. 点击页面右上角的「导出」按钮
  3. 在弹出的导出对话框中：
     - 选择日期范围：例如「本月」或自定义「2025-01-01 至 2025-01-31」
     - 选择导出范围：daily（按天汇总）/ daily_with_keys（按天+密钥）/ daily_with_models（按天+模型）
     - 选择导出格式：CSV 或 JSON
  4. 点击「确认导出」

  演示者：导出的文件包含了以下字段：
  | 字段 | 说明 |
  |------|------|
  | Date | 日期 |
  | Team / Key / User | 实体名称 |
  | Entity ID | 实体 ID |
  | Spend ($) | 消费金额 |
  | Requests | API 请求次数 |
  | Successful Requests | 成功请求数 |
  | Failed Requests | 失败请求数 |
  | Total Tokens | 总 Token 数 |
  | Prompt Tokens | 输入 Token |
  | Completion Tokens | 输出 Token |
